# Docs

This folder contains the narrative documentation for the submission.

## Recommended Reading

- `project_summary.md`: high-level summary of the problem, final deliverable,
  main results, and interpretation.
- `reproducibility.md`: checklist for what a reproducible run folder must
  contain.
- `original/`: copied notes, status files, and historical documentation from
  the original development workspace. Use these as audit trail, not as the
  primary run instructions.

## What To Do Next

Use `project_summary.md` to understand the research conclusion, then use the
top-level `README.md`, `configs/README.md`, and `src/README.md` to run the
current semi-auto pipeline. Use `experiments/best_current/README.md` when
deciding the next full-auto localization experiment.
